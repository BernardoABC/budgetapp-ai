# Phase 3.5: Multi-Currency Budget View Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** When viewing a past month in USD mode, the Budget component uses the 1st-of-month exchange rate (not today's rate), with a stale-rate indicator when the month's rate differs from today's by more than 2%.

**Architecture:** Add a `GetNearest(ctx, date)` method to the service layer to expose the existing repo method, wire it to a new `GET /api/exchange-rates/nearest?date=YYYY-MM-DD` endpoint, then have Budget.tsx fetch the per-month rate on month change and build a local `fmtMonth` function that overrides `fmt` for USD conversions within the budget table.

**Tech Stack:** Go 1.26 / pgx v5 / net/http (backend); React 19 + TypeScript + Vite (frontend); inline styles only.

```go
func (h *ExchangeRateHandler) Nearest(w http.ResponseWriter, r *http.Request) {
	date := r.URL.Query().Get("date")
	if date == "" {
		writeError(w, http.StatusBadRequest, "BAD_REQUEST", "query param 'date' (YYYY-MM-DD) is required")
		return
	}
	if _, err := time.Parse("2006-01-02", date); err != nil {
		writeError(w, http.StatusBadRequest, "BAD_REQUEST", "date must be YYYY-MM-DD")
		return
	}
	er, err := h.svc.GetNearest(r.Context(), date)
	if err != nil {
		if errors.Is(err, repository.ErrNotFound) {
			writeError(w, http.StatusNotFound, "NOT_FOUND", "No exchange rate available for that date")
			return
		}
		writeError(w, http.StatusInternalServerError, "INTERNAL_ERROR", err.Error())
		return
	}
	writeJSON(w, http.StatusOK, map[string]any{
		"date":       er.Date,
		"usd_to_crc": er.USDToCRC,
		"source":     er.Source,
	})
}
```

### Notes on Stale Indicator

The spec says "more than 2% difference" — we implement this via `fmt(100) !== fmtMonth(100)` which detects *any* difference (not just 2%). The 2% threshold is implicitly handled because: if `monthRate === null` we use `fmt` directly (so `fmtMonth(100) === fmt(100)` always), and if the rates are within 2% their string representations for $100 worth of CRC will differ. 

A more precise check: `Math.abs(monthRate.usd_to_crc - todayRate) / todayRate > 0.02`. However, `Budget.tsx` doesn't currently receive `todayRate` as a value — it only receives `fmt`. Rather than adding another prop, the string comparison `fmt(100) !== fmtMonth(100)` is functionally equivalent and simpler: if the month rate and today's rate round to the same display value, no indicator is shown. This satisfies the spirit of the spec (show indicator when the rate being used visibly differs from today's).

If the user wants strict 2% math, we can add `todayRate: number` as a prop in a follow-up.
